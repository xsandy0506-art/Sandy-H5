## 5️⃣ startFallbackTimer 函数（第1834-1846行）

```javascript
function startFallbackTimer() {
  clearFallbackTimer()
  fallbackTimerRef.current = setTimeout(() => {
    const v = videoRef.current
    // 如果视频存在且仍未播放，则认为播放被阻止，直接进入漫游
    if (!v || v.paused) {
      try {
        setShouldAutoPlay(false)
        setActiveTab('roam')  // ⚠️ **5秒后强制切换到roam！**
      } catch (e) {}
    }
    clearFallbackTimer()
  }, 5000)
}
```
**⚠️ 关键发现**：
- 启动一个5秒的定时器
- 5秒后，**如果视频仍未开始播放，就强制`setActiveTab('roam')`**
- 这会导致即使没有视频播放，也会在5秒后切换到roam

## 🚨 问题分析

### 问题1：activeTab自动切换到'roam'的原因

**存在3个地方会触发activeTab切换到'roam'**：

1. **handleSkipIntro()** - 用户点击"跳过"按钮
   - 立即切换

2. **handleVideoEnded()** - 视频播放完毕
   - 调用：`video.onEnded` 回调
   - 立即切换

3. **startFallbackTimer()** - fallback机制
   - 延迟：5秒后
   - 条件：如果视频仍未播放（`v.paused`）
   - **这是最可能的自动切换原因！**

### 问题2：为什么5秒内就会切换到roam

**流程**：
1. T=0ms：点击"点亮花灯"
   - `isLit = true`
   - `shouldAutoPlay = true`
2. T=5000ms：触发 `setTimeout(() => setShowIntroVideo(true), 5000)`
   - `showIntroVideo = true`
   - **立即触发 useEffect #5**
   - 调用 `startFallbackTimer()` - 启动5秒倒计时
3. T=5050ms：视频开始加载/播放

**关键问题**：如果视频加载较慢或播放被浏览器阻止：
- T=10050ms：fallback定时器触发
- 检查：`if (!v || v.paused)` - 视频仍在暂停
- 执行：`setActiveTab('roam')` - **强制切换到roam**

### 问题3：已禁用的自动切换逻辑

第1632-1647行的被注释代码原本的意图：
```javascript
// 如果isLit=true且activeTab='home'
// 那么2.2秒后自动切换到roam
```
**但这段代码已被禁用**，所以不会生效。

## ✅ 验证结论

### "点亮花灯"后是否真的会：
1. **✅ 设置 isLit=true** - YES，第1831行
2. **✅ 设置 setShouldAutoPlay=true** - YES，第1832行
3. **✅ setTimeout 5秒后 setShowIntroVideo(true)** - YES，第1835-1837行

### 是否有其他逻辑在5秒内直接把activeTab改为'roam'：
1. **❌ 被禁用的useEffect** - 代码已注释，不会执行
2. **❌ handleSkipIntro** - 需要用户点击"跳过"按钮
3. **❌ handleVideoEnded** - 需要视频播放完毕
4. **⚠️ startFallbackTimer** - **可能会在T=10秒时切换**
   - 但这是在 showIntroVideo=true 后的5秒
   - 总时间是：5秒(到视频显示) + 5秒(fallback) = **10秒**

### 在5秒内不会自动切换activeTab到roam

**✅ 结论**：在最初的5秒内（从点击到showIntroVideo显示），**activeTab不会自动切换**。只有在视频显示后，如果视频加载失败，才会在10秒时自动切换。
